## 1. 欧盟发布 AI 系统透明度义务指南

欧盟委员会发布《人工智能法案》第 50 条透明度指南，帮助模型提供者和部署者准备自 2026 年 8 月 2 日起适用的要求。指南明确交互式 AI 应告知用户其正在与 AI 交流，生成或操纵内容需带机器可读标记，部署者还须披露深度伪造、未经人工编辑的公共利益内容、情绪识别和生物特征分类。

来源：https://digital-strategy.ec.europa.eu/en/news/commission-publishes-guidelines-transparency-obligations-providers-and-deployers-certain-ai-systems

## 2. Machine Genome 为模型与 Agent 建立可验证谱系

Machine Genome 是面向模型、Agent、Harness、数据集及其产物的开放身份和来源协议，以规范化创世记录的 SHA-256 多重哈希生成稳定 Gene。协议用 Ed25519 DID 签名、类型化父边、追加式修订与第三方证明构建谱系图，注册表还提供 Merkle 包含证明、检查点和冲突证据，但明确不把签名等同于能力或所有权证明。

来源：https://github.com/paxlabs-inc/machine-genome

## 3. Nativ 在 macOS 统一运行和服务 MLX 多模态模型

Nativ 将 mlx-vlm Server、模型发现、聊天、日志和性能监控封装进原生 SwiftUI 应用，可在 Apple Silicon 上本地处理文本与图片。它暴露 OpenAI Chat/Responses、图片、音频和模型兼容端点以及 Anthropic Messages 接口，并能把 Codex、Claude Code、Pi、Hermes 和 OpenCode 指向本机模型。

来源：https://github.com/Blaizzy/nativ

## 4. Murmur 用 Apple Intelligence 和 Whisper 做全本地听写

Murmur 在 macOS 上提供按键说话输入，可选择系统 SpeechAnalyzer 或基于 WhisperKit 的 Core ML Whisper 引擎，音频与转录均留在设备。它会从人工纠正中学习发音和词汇，并用本地 Apple Intelligence 做按应用语气改写、语法润色与提示词结构化。

来源：https://github.com/janisbelozerovs-dev/murmur

## 5. Open Dictate 面向繁体中文建立可审查语音闭环

Open Dictate 用 MLX Whisper 在 Apple Silicon 上本地完成繁体中文听写和会议转录，再应用用户维护的确定性词库校正。工具把可能误听项放进审核队列而不静默改写含义，并规划以匿名说话者标签、个人词库和本地声纹资料持续改进后续识别。

来源：https://github.com/frank890417/open-dictate

## 6. STEM Illustration Skill 提供 24 类科研配图模板

STEM Illustration Skill 覆盖信号通路、实验流程、细胞结构、机制图、质粒图、机器学习架构和学术海报等 24 个场景，并可在学术、教材、信息图和 3D 渲染风格间切换。项目明确禁止生成虚构实验数据，严格几何改用代码绘制，还用错误化学反应样例提醒用户核验 AI 图像中的科学内容。

来源：https://github.com/liangdabiao/stem-illustration-skill

## 7. 沐阳手绘视频从一句主题生成完整动画讲解

沐阳手绘视频 Skill 将脚本、分镜、逐幕 AI 插图、连续旁白、同步字幕、Remotion 动效与 MP4 验证串成一条流水线。它提供十种手绘视觉风格，可选择 Fish Audio 或 ElevenLabs 生成配音，并输出可继续修改图片、字幕、动画和字体的 Remotion 工程。

来源：https://github.com/yokel1121/muyang-handdrawn-video

## 8. Oil Visual 为 Codex 定义一致的漫画墨线配图语言

Oil Visual 用粗黑墨线、半调网点、米白纸张、固定暖黄色和有限语义色约束 Codex 生成的解释性插图。规范还固定圆框眼镜小人与边牧角色，并要求图片内文字短而准确，使流程、对比、循环和瓶颈等不同主题保持统一视觉身份。

来源：https://github.com/oil-oil/oil-visual

## 9. Open Kritt 用并行 Agent 编排代码安全研究

Open Kritt 将整库安全审计拆成多个聚焦任务，由 Codex 或 Claude Code 并行分析本地或远程仓库，再对发现去重、排序和验证。平台支持可复用工作流、后处理脚本、概念验证与自定义严重度规则，同时提醒工具型 Agent 在一次性容器中拥有 Root 和网络权限，应在专用主机或虚拟机上扫描不可信代码。

来源：https://github.com/Kritt-ai/open-kritt

## 10. Clodex 将 OpenAI 模型桥接到 Claude Code

Clodex 把 OpenAI API 或 ChatGPT/Codex 计划中的模型转换为 Claude Code 可使用的模型，并支持主会话、子 Agent、工作流和 Agent 团队。它提供选择性代理与本地兼容网关两种模式，可保存模型别名、真实上下文窗口和系统凭据库中的 OAuth Token，但可选二进制补丁需在 Claude Code 更新后重新应用。

来源：https://github.com/bman654/clodex

## 11. Sole Search 用官方来源筛选韩国小商户扶持项目

Sole Search 是面向 Claude Code、Codex、Antigravity、Cursor 和 Gemini CLI 的调查 Skill，可抓取韩国小商户官方平台当前公告，并按行业、地区、经营年限、员工和营收执行五阶段资格判断。输出按截止日期列出可立即申请项目、判定依据和覆盖清单，把官方公示页与附件作为最终证据。

来源：https://github.com/djfksjd/sole-search

## 12. Universal API Reference 统一记录三千多个应用接口

Universal API Reference 用一致、可搜索的格式整理 3113 个应用和 74880 个端点，逐项记录认证、分页、过滤、排序、参数、响应与原生 API 映射。仓库提供机器可读目录、OpenAPI 索引和 `llms.txt`，方便 Agent 或 RAG 管线按应用加载上下文，并以每日提交同步生产规格变化。

来源：https://github.com/mindcloud-inc/universal-api-reference

## 13. Pi For Each 用隔离会话执行顺序提示循环

Pi For Each 为 Pi Agent 增加 `/for` 和 `$each@路径` 语法，可逐个遍历目录子项或文件行，而无需把循环控制结构告诉模型。首轮留在当前会话，后续轮次从相同早期上下文分别分叉并替换迭代提示，从而避免前一轮结果污染后续任务，同时保留严格顺序执行和用户可见进度。

来源：https://github.com/jejay/pi-for-each

## 14. Ailin Collective Intelligence 编排多模型协作与审计

Ailin Collective Intelligence 通过多种辩论、评审、仲裁和综合策略协调不同提供商与开放权重模型，而不只是为单次请求选择一个模型。系统记录每次模型选择与协调决策，提供成本上限、租户隔离、降级回退和提供商故障绕行，并在文档中区分已投入生产与仍在路线图上的模块。

来源：https://github.com/ailinone/collective-intelligence

## 15. LLM-WIKI 将非结构化文章编译成知识节点

LLM-WIKI 把文章、教程和观点拆成带标准名称、定义、显式关系、推理链与短版 RAG 内容的本地 Markdown 节点。它还能从跨领域节点生成“发芽报告”，或围绕主题多轮检索后进行融合写作，使个人知识库更适合模型检索、引用和组合推理。

来源：https://github.com/loonggg/LLM-WIKI
