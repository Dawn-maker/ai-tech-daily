# AI 技术热点日报｜2026-07-12

筛选口径：只收录北京时间 2026-07-12 00:00-23:59 发布、创建或实质更新的 AI 技术内容。技术新闻优先，关注模型、API、开源项目、论文进展、AI agent、图像/视频/多模态生成、算力基础设施、开发者工具和安全评估。每条 `来源` 都指向具体页面。

## 1. Clodex：强调受控执行与可验证证据的本地优先 Agent IDE

Clodex 在 GitHub 创建技术预览版，把持久任务、代码、终端、浏览器、Git、模型路由和记忆整合进 Electron 工作区。其核心设计把模型输出视为不可信输入，通过权限策略、隔离运行时、人工审批、审计记录和可回看证据约束自治执行，面向长时间运行的软件工程 Agent。

来源：https://github.com/mereyabdenbekuly-ctrl/clodex-ide

## 2. jlens-gguf：为 llama.cpp/GGUF 模型加入交互式 Jacobian Lens

jlens-gguf 将 Jacobian Lens 适配到 GGUF 模型和 llama.cpp，不依赖 PyTorch 即可读取残差流并完成可视化。项目支持实时 steering、概念替换、消融和受控生成，为本地量化模型的机制解释和中间表示干预提供了直接工具链。

来源：https://github.com/igorbarshteyn/jlens-gguf

## 3. audio.cpp-webui：为纯 C++ 本地音频推理补齐完整 WebUI

该项目基于 ggml 的 audio.cpp，新增 Windows 友好的 WebUI、启动脚本和 OpenAI 兼容服务接口。它把 TTS、ASR、VAD、声音转换、说话人分离及音乐生成统一到无需 Python 运行时的本地界面中，并支持按需加载和切换模型。

来源：https://github.com/kigner/audio.cpp-webui

## 4. Headroom：统一监控并调度 Claude 与 Codex 使用额度

Headroom 提供本地实时仪表板，用于查看多个 Claude 和 ChatGPT/Codex 账户的额度窗口，并在账户达到限制后切换到仍有余量的账户。项目通过已有 CLI 的额度读取接口获取状态，不消耗模型 token，面向长时间运行的编码 Agent 工作流。

来源：https://github.com/domanski-ai/headroom

## 5. Kimchi：用 13 个产品角色把模糊想法整理成可执行文档

Kimchi 是面向 Claude Code 的产品思考插件，将原始创意依次交给产品、用户研究、技术、设计和风险等 13 个角色分析。输出包括可直接交付给编码 Agent 的需求、架构、路线图和验收材料，试图把 vibe coding 前置为结构化产品规划。

来源：https://github.com/chitransh-cj/kimchi

## 6. Plandeck：为长时间运行的 AI Agent 提供可视化任务看板

Plandeck 把 Agent 的计划映射为带依赖关系的 Kanban 看板，自动标示 Ready、Blocked 和关键路径。它针对长任务中 Markdown 计划难以快速判断下一步的问题，让用户可以从可视界面监督任务解锁与执行进度。

来源：https://github.com/OthmanAdi/plandeck

## 7. pocket_disasm：让编码 Agent 并行操作多个 IDALib 逆向会话

pocket_disasm 是一个多会话 IDALib MCP 路由器，可让编码 Agent 同时分析多个二进制文件。它暴露接近 IDA 的反汇编和逆向能力，并将各会话隔离管理，适合自动化漏洞研究、样本对比和跨模块追踪。

来源：https://github.com/whoisqwerz/pocket_disasm

## 8. LLM Inference Optimization Lab：可复现的 CPU 推理优化实验台

该项目围绕 llama.cpp 构建 CPU LLM 推理的基准、性能剖析和热点内核优化流程。它固定依赖与工作负载，保留原始结果、二进制来源和统计区间，并明确区分微基准提升与端到端收益，方便复核量化格式和本地优化结论。

来源：https://github.com/200lz/llm-inference-optimization-lab

## 9. Korean Bill & Debate MCP：用官方记录串联韩国国会法案与辩论

Korean Bill & Debate MCP 将韩国国会开放 API、法案状态、委员会会议记录和审查报告接入自然语言调查流程。系统不仅返回搜索结果，还连接法案、会议、人物、发言和答复，并附官方原文位置，强化 Agent 调研的可验证性。

来源：https://github.com/epoko77-ai/korean-bill-debate-mcp

## 10. Recensa：本地检索、回放和审计 Claude Code 会话

Recensa 将 `~/.claude/projects/` 下的 Claude Code JSONL 会话索引到本地 SQLite，并提供三栏式 Web 阅读器。用户可搜索、回放和审计对话、工具调用及子 Agent 活动，原始记录保持只读且不上传，适合复盘长任务和排查 Agent 行为。

来源：https://github.com/S40911120/recensa

## 11. Kit：在 macOS 菜单栏中离线完成语音识别与翻译

Kit 把截图、录屏、录音、转写、翻译和实时口译集中到 macOS 菜单栏应用。其 AI 功能使用本地 Whisper 与 Argos 等组件，无需 API key 或联网即可处理音频和多语言文本，为轻量多模态创作与沟通提供一体化桌面工具。

来源：https://github.com/commme/kit

## 12. SO-101 Bench × GR00T-N1.7：无需实体机器人评测视觉语言动作模型

该项目把 SO-101 桌面操作基准迁移到 NVIDIA GR00T-N1.7，并在 Isaac Lab 数字孪生环境中完成微调和评测。它覆盖 56 种物体及多类语言条件操作任务，使用 Cosmos-Reason2/Qwen3-VL 视觉语言骨干，让开发者无需实体机械臂也能验证 VLA 策略。

来源：https://github.com/kabilankb/so101-bench-groot-n1.7

## 13. ears：基于个人声学基线理解说话语气

ears 从音高、音量、停顿、语速和颤动中提取声学特征，并用用户自己的历史中位数与 MAD 建立个性化基线。它把转写文本、情绪判断和相对声学变化一起提供给 AI 伴侣，避免仅依赖平均人群标签判断同一句话的真实语气。

来源：https://github.com/eveacla11/ears

## 14. Stinger：用本地诱饵检测 Agent 与软件窃取凭据

Stinger 在 macOS 和 Linux 工作站中放置 FIFO、套接字等诱饵，记录进程对密钥、令牌和浏览器数据收集路径的触碰。它还可在伪造 HOME 与精简环境中运行命令或编码 Agent，并在命中陷阱时延迟、拒绝或终止进程范围，为低权限端点提供实验性防护。

来源：https://github.com/0x4D31/stinger
