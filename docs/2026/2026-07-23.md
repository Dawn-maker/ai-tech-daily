## 1. ChatGPT Health 接入医疗记录与 Apple Health

OpenAI 开始向美国成年用户推出 Health，允许在获得许可后连接 Apple Health、医院医疗记录、One Medical 和 Function Health，并在普通对话中调用药物、化验、就诊、睡眠与活动等上下文。连接数据及使用这些数据的对话不会用于训练基础模型或定向广告，默认按次请求授权；GPT-5.6 系列在 HealthBench Professional 上也全面超过 GPT-5.5，但产品仍明确要求重要信息由专业医护人员核验。

来源：https://openai.com/index/health-in-chatgpt/

## 2. Diffusers 原生支持 Nunchaku 4-bit 扩散推理

Hugging Face 将 Nunchaku 的 SVDQuant 推理路径集成进 Diffusers，预量化模型现在可直接通过 `from_pretrained()` 加载，并借助 kernels 包免去本地 CUDA 编译。其 W4A4 路径把异常激活移入低秩 16-bit 分支，再融合低秩投影、量化和 4-bit 矩阵乘；示例中的 ERNIE-Image-Turbo 在 RTX 5090 上生成 1024×1024 图像约需 1.7 秒，峰值显存约从 BF16 的 24GB 降至 12GB。

来源：https://huggingface.co/blog/nunchaku-diffusers

## 3. GitHub Issues 为 Agent 自动化加入审批、置信度与理由

GitHub Issues 公测 Agent 自动化控制，可让 Agent 对标签、字段、类型、关闭和指派等操作附带高、中、低置信度和可审计理由，并按仓库阈值自动执行或进入人工审批面板。能力同时适配 GitHub Agentic Workflows、Copilot cloud agent、REST 与 GraphQL API，但 GitHub 明确说明审批只是工作流便利，不构成服务端安全边界。

来源：https://github.blog/changelog/2026-07-23-agent-automation-controls-in-github-issues-in-public-preview/

## 4. GitHub Mobile 可让 Copilot 直接修复失败的 CI

来源：https://github.blog/changelog/2026-07-23-github-mobile-fix-failing-actions-checks-with-copilot-cloud-agent/

## 5. Discourse AI 自动生成可搜索的多语种图片说明

Discourse AI 将图片说明改为发帖后后台自动生成，并把说明与作者原始正文分离存储，供屏幕阅读器、图片灯箱和站内搜索使用。管理员可配置具备视觉能力的说明 Agent，也可限速回填旧帖；生成结果支持编辑和本地化，不会以可见正文或传统 alt 文本的形式写回帖子。

来源：https://meta.discourse.org/t/making-post-images-more-accessible-with-discourse-ai/408323?tl=en

## 6. 美联社更新生成式 AI 与编码助手使用规范

美联社允许记者用 AI 辅助早期研究、文档摘要、转录、翻译、标题建议和文字校正，但所有输出必须经记者审阅，采编判断、核验与责任仍由人承担。新版规范继续禁止生成或改动新闻摄影，要求对新闻中的合成内容和实质参与成稿的生成式 AI 明确披露，并首次加入软件开发中使用 AI 编码助手的指导。

## 7. 2890 万参数语言模型在 8 美元 ESP32-S3 上离线运行

esp32-ai 展示一个 2890 万参数、4-bit 后约 14.9MB 的语言模型，在只有 512KB SRAM、8MB PSRAM 和 16MB Flash 的 ESP32-S3 上以约 9.5 Token/s 生成文本，全程无需联网。项目借鉴 Gemma 的 Per-Layer Embeddings，把约 2500 万参数的查找表留在 Flash，每个 Token 只取少量行，而把持续计算的核心放在高速内存中。

来源：https://github.com/slvDev/esp32-ai

## 8. Vinv 用真实运行轨迹独立验证编码 Agent 的修复

Vinv 为 Python 后端记录请求、函数调用、时延、内存、参数、返回值和错误，再把运行证据叠加到本地语义索引与代码图，通过 MCP 提供给现有编码 Agent。Agent 声称完成修复后，Vinv 会重启服务、回放流量并运行对 Agent 隐藏的验收测试，形成“发现—修复—独立验证”的闭环；TypeScript 和 Go 当前仅支持索引与探测，尚无运行时追踪。

来源：https://github.com/VinvAI/VinvAI

## 9. AxisAgentic 开源长时程 Agent 运行与评测框架

AxisAgentic 将追加式对话轨迹、工具编排、上下文压缩、回滚重试、自验证、指标记录和训练数据导出组合成可扩展运行时，并把通用基础设施与任务配方、数据集和评测器分离。首个 XYZ-Aquila 搜索配方支持 OpenAI 兼容端点与本地模型客户端，能够精确重建每次模型调用可见的上下文，并导出可回放的 SFT 轨迹。

来源：https://github.com/XYZ-AI-Lab/AxisAgentic

## 10. Auto-BenchMax 开源面向 Agent 基准的数据合成流水线

Auto-BenchMax 读取工具调用或 Agent 基准的评测入口，按评分器类型选择“执行后提取”或“构造后验证”流程，生成同分布轨迹并筛选成训练数据。项目公开 MCP-Atlas 环境、数据和训练代码，报告首轮数据把基础模型公开集得分从 19.1 提升到 40.4，同时也让方法可以针对其他规则评分或 LLM 裁判基准复现。

来源：https://github.com/sunny-glow/Auto-BenchMax

## 11. 三台 DGX Spark 部署 GLM-5.2 获得约 24.8 万上下文

来源：https://github.com/MiaAI-Lab/GLM-5.2-NVFP4-AQLM-Triple-DGX-Sparks
