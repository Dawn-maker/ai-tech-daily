# AI 技术热点日报｜2026-07-09

## 1. OpenAI 正式发布 GPT-5.6 模型家族

OpenAI 在 7 月 9 日发布 GPT-5.6，把模型家族明确拆成 Sol、Terra 和 Luna 三档，继续按旗舰、低成本和轻量化场景分层。它的定位不只是一次常规升版，而是把高强度推理、长任务执行和更广的产品接入统一到同一代底座上。对模型应用团队来说，这意味着 2026 年下半年的 OpenAI 产品线将更多围绕 5.6 系列展开。

来源：https://openai.com/index/gpt-5-6/

## 2. OpenAI 同步公开 GPT-5.6 System Card

OpenAI 在 Deployment Safety Hub 上同步发布 GPT-5.6 System Card，覆盖模型数据与训练、视觉能力、越狱、提示注入、健康场景和 computer use 确认机制。文档还给出了基于部署模拟的违规内容预测、外部机构对 cyber 与 alignment 的评估结果，以及对“reasoning 可监控性”的最新判断。对需要做高风险场景接入的团队来说，这份系统卡比单纯发布博客更有参考价值。

来源：https://deploymentsafety.openai.com/gpt-5-6

## 3. GPT-5.6 成为 Microsoft 365 Copilot 的首选模型

OpenAI 同日宣布 GPT-5.6 已成为 Microsoft 365 Copilot 的 preferred model，说明这代模型并非只停留在实验性入口，而是直接进入企业办公主产品。对企业用户来说，这通常意味着文档生成、会议总结和工作流协作能力会随底模升级一起提升。对生态观察者来说，这也显示 OpenAI 与微软的联合分发仍是新模型落地的核心渠道。

来源：https://openai.com/index/gpt-5-6-preferred-model-microsoft-365-copilot/

## 4. OpenAI 推出 ChatGPT Work，把 agent 能力扩展到跨应用执行

OpenAI 7 月 9 日推出 ChatGPT Work，强调它可以跨应用和文件搜集信息、持续数小时处理复杂项目，并产出 slides、docs、sheets 甚至 web app。官方还明确表示 ChatGPT Work 由 GPT-5.6 驱动，并继承了 Codex 的执行式工作流能力。对 AI 办公和企业 agent 赛道来说，这标志着“问答助手”正在进一步演化为可交付成品的任务代理。

来源：https://openai.com/index/chatgpt-for-your-most-ambitious-work/

## 5. OpenAI 把 Bio Bug Bounty 升级为长期项目并提高奖金

OpenAI 把原有 GPT-5.5 生物安全红队测试扩展为持续性的 OpenAI Bio Bug Bounty Program，并把通用越狱奖励提高到 5 万美元。新计划从 GPT-5.6 开始持续覆盖前沿模型，目标是专门测试生物风险相关的通用 jailbreak。对安全评估社区来说，这说明 frontier 模型的生物安全对抗测试开始从一次性活动转向制度化项目。

来源：https://openai.com/index/bio-bug-bounty/

## 6. OpenAI Python SDK 2.45.0 发布，率先接入 GPT-5.6 Sol 更新

`openai-python` 在 7 月 9 日发布 `v2.45.0`，主要特性是加入 GPT-5.6 Sol 的 API 更新，并修复 beta 资源访问器。虽然这是 SDK 层的小版本发布，但它通常是模型能力进入开发者主线环境的最早信号之一。对 API 集成团队来说，这意味着 5.6 相关能力已开始进入正式开发工具链。

来源：https://github.com/openai/openai-python/releases/tag/v2.45.0

## 7. Anthropic 发起“Hard Questions”计划，公开征集最难的 AI 问题

Anthropic 在 7 月 9 日发布 `Inviting hard questions`，把公众对 AI 的担忧、期望和治理问题正式纳入一个长期公开议题。它把重点放在失业、创作价值、人类能动性、生物与安全风险，以及 AI 是否真正带来公共利益等硬问题上。对 AI 公司治理而言，这代表厂商正尝试把公众意见收集、进度透明化和产品责任叙事结合起来。

来源：https://www.anthropic.com/news/hard-questions

## 8. Anthropic 上线 Claude 使用反思面板

Anthropic 同日推出 beta 版 `reflect` 功能，让用户在 Claude 网页端和桌面端查看主题分布、使用模式与时间跨度，并设置 quiet hours 或休息提醒。这个功能并不直接提升模型能力，但它把“如何使用 AI”本身做成了产品内反馈回路。对 AI 产品设计来说，这是一种把使用透明度、行为引导和用户自控整合进助手界面的新尝试。

来源：https://www.anthropic.com/news/reflect-with-claude

## 9. Meta 推出 Muse Spark 1.1，加入 AI 编码竞赛

来源：https://techcrunch.com/2026/07/09/meta-enters-the-crowded-ai-coding-battle-with-muse-spark-1-1/

## 10. Ollama 用户规模逼近 900 万，继续强化本地运行大模型路线

## 12. llama.cpp 当天连续发版，修复 chat template “thinking probe” 崩溃问题

`llama.cpp` 在 7 月 9 日连续发布多个构建，其中 `b9945` 重点修复了 server 侧 chat template 在启用 thinking probe 时可能导致 `llama-cli` 直接中止的问题。修复方式是把相关探针逻辑移入统一的 `try/catch` 初始化路径，避免 apply 时抛出未捕获异常。对本地推理开发者来说，这类底层稳定性改动比表面功能更关键，因为它直接影响模型加载和服务可用性。

来源：https://github.com/ggml-org/llama.cpp/releases/tag/b9945
