# AI 技术热点日报｜2026-07-19

筛选口径：只收录北京时间 2026-07-19 00:00-23:59 发布、创建或实质更新的 AI 技术内容。技术新闻优先，关注模型、API、开源项目、论文进展、AI agent、图像/视频/多模态生成、算力基础设施、开发者工具和安全评估。每条 `来源` 都指向具体页面。

## 1. Codor 用统一频道连接多个编码 Agent

Codor 在本机启动后台服务，把 Claude Code、Codex、Gemini、Copilot 和 OpenCode 会话接入同一频道，并支持把正在运行的会话直接加入而不丢失上下文。项目通过一次性浏览器配对与私有 Token 控制访问，也可用 Tailscale 将本地界面安全地开放给手机或其他设备，当前处于 Alpha 阶段。

来源：https://github.com/rjx18/codor

## 2. Mentor 从本地会话历史生成编码 Agent 使用报告

Mentor 读取本机 Claude Code 与 Codex 日志，统计会话、消息、工具调用、编辑、Token 和语言分布，再生成无需外部依赖的 HTML 报告。报告将重复返工、错误习惯和未充分使用的能力转成具体工作流建议，并明确把主题归纳等模型判断标为假设而非事实。

来源：https://github.com/smixs/mentor

## 3. Conformer STT 在 ESP32-S3 上运行 1300 万参数语音识别

Conformer STT S3 将蒸馏和量化后的 1310 万参数 Conformer 部署到 ESP32-S3，使英语语音可在微控制器本地转成小写文本。项目给出多个公开语音集的词错误率、不同开发板的推理耗时和内存占用，并用简单声音活动检测限制最长输入以适配 SRAM 与 PSRAM。

来源：https://github.com/lspr98/conformer-stt-s3

## 4. LiTranslate 用双 Agent 完成视频字幕与本地化

LiTranslate 在浏览器中用 ffmpeg.wasm 提取并切分音频，再由独立转写 Agent 生成带时间轴字幕、翻译 Agent 按一对一行映射本地化文本。后台任务把进度持续写入 SQLite，刷新页面后仍可恢复，并提供字幕时间线编辑以及 SRT、VTT、ASS 和 TXT 导出。

来源：https://github.com/AliAkrami1375/Li-Translate

## 5. OpenBrowser 集成隔离浏览器、RPA 与 MCP

OpenBrowser 在桌面端管理相互隔离的 Chromium 环境，并为每个环境配置代理、语言、时区、User-Agent、Canvas、WebGL 和扩展。它还提供基于 CDP 的窗口同步、本地 RPA 流程以及默认仅监听回环地址的 API/MCP 接口，支持 Windows 与 macOS。

来源：https://github.com/lyu0805/OpenBrowser

## 6. Agentvis 比较不同编码 Agent 的成本与代码影响

Agentvis 读取 OpenClaw、Claude Code、Codex 和 Hermes 的本地 JSONL 记录，按公开 API 价格估算成本并重建代码增删。仪表盘可比较每次编辑成本、缓存效率、工具错误率和延迟，还会识别同文件反复修改、纠正性追问与无最终回复的会话，且默认只读并绑定本机。

来源：https://github.com/IAmUnbounded/Agentvis

## 7. Software Periodic Table 为编码 Agent 建立软件原子库

Software Periodic Table 将常见业务应用抽象为对象、属性、动作、界面、智能与规则六类共 115 个元素，并提供严格类型的 TypeScript 参考实现。其组合框架让 Agent 优先检索和连接已验证原子，而非每次重新生成模式，仓库还附带评测工具比较 Token 效率、结构忠实度与正确性。

来源：https://github.com/NullLabTests/software-periodic-table

## 8. Graphkit 用独立上下文监督长时编码任务

Graphkit 把长任务拆成执行节点和定期启动的干净上下文监督节点，两者只通过账本、Git 状态和单向指令文件交换信息。执行者每轮只处理一个条目并立即验证，监督者从外部检查范围漂移、虚假完成和指标退化，从而允许日常执行使用较便宜模型而把高成本判断集中在周期审查。

来源：https://github.com/levi-qiao/graphkit

## 9. Phosphene 用 MCP 把 AI 陪伴转成可审计日常任务

Phosphene 允许 AI 通过七个 MCP 工具创建日常、挑战和惊喜任务，用户在 PWA 中提交文字或图片证据并积累积分、连击与成就。系统提供 AI 审核与自我确认、不可变积分账本、图片去 EXIF/GPS、Token 轮换、CSRF 防护和完整审计日志，数据以 SQLite 与私有文件保存在单一持久卷。

来源：https://github.com/3lmglow/Phosphene

## 10. SOMCP 在 Android 手机上提供原生 SO 逆向工具

SOMCP 是 Android 前台服务，通过 Streamable HTTP 向 Agent 暴露 ELF 分析、Rizin 反汇编、LIEF 修复、补丁会话和构建导出。它以精简工具清单降低初始化上下文，并提供快照、回滚、撤销、审计、全局并发闸门与 Token 访问控制，客户端还可经 ADB 端口转发在电脑侧调用。

来源：https://github.com/bilieebiliee1-design/SOMCP

## 11. SparkDash 统一监控多台 DGX Spark 的本地推理

SparkDash 在单一 Web 仪表盘中实时展示多台 NVIDIA DGX Spark 的 GPU、CPU、统一内存、存储、网络和本地 LLM 指标。它通过 WebSocket 推送快照，可自动探测 llama.cpp、vLLM 或 SGLang 的吞吐，并支持本机采集或 SSH 远程采集，SSH 密码使用 AES-256-GCM 加密。

来源：https://github.com/MiaAI-Lab/sparkDash

## 12. Relay Between Agents 设计可审计的规划—施工通道

Relay Between Agents 用结构化任务表、授权等级和交付收据连接规划 AI 与编码 Agent，避免人类反复复制需求和报告。教程以 Google Sheet、Drive 与本地桥接为参考实现，要求高影响操作暂停等待确认，并让规划端能够直接读取证据、验收结果与恢复说明。

来源：https://github.com/Gwendolenmave/relay-between-agents

## 13. Claude Code Custom Prompt 给出失效关闭的提示词替换流程

该工具包用包装脚本按编号拼接 Markdown 文件，并通过 `--system-prompt-file` 替换 Claude Code 主系统提示词，同时区分运行时工具描述与内置压缩模板。对于需要修改二进制的 `/compact` 模板，它要求固定版本、唯一锚点计数、隔离暂存、反向解包和真实压缩验证，任何结构不匹配都拒绝生成可安装产物。

来源：https://github.com/rossignol6712/claude-code-custom-prompt
