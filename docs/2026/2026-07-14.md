## 1. Google 公布 Qwen3.5-397B 在 Ironwood TPU 上的系统优化方案

Google 用 JAX/Pallas 内核、Attention 数据并行与专家并行混合拓扑优化 Qwen3.5-397B-A17B，在 512 并发下把偏解码和偏预填充工作负载分别提速约 3.1 倍与 4.7 倍。方案还融合 Ragged Page Attention、Gated DeltaNet 与分层 Reduce-Scatter，使实际吞吐接近 TPU v7x 的计算和 HBM 带宽 roofline。

来源：https://developers.googleblog.com/systems-engineering-playbook-optimizing-qwen-35-397b-moe-on-ironwood-tpu7x/

## 2. PrismML 发布 1-bit 多模态模型 Bonsai 27B

PrismML 推出基于 Qwen3.6 27B 的 Bonsai 27B，提供 1-bit 二值和 1.58-bit 三值两种低比特版本，支持图像输入、推理、结构化工具调用与计算机操作 Agent。官方称 1-bit 版本内存占用约 3.9GB，三值版本约 5.9GB，目标是在手机和普通笔记本上运行 27B 级多模态能力。

来源：https://prismml.com/news/bonsai-27b

Google Cloud 介绍了通过 Vertex AI Model Garden 和 Agent Platform 部署 Claude 的生产架构，支持区域端点、容量管理、监控与跨区域路由。开发者可使用 AnthropicVertex SDK 发起调用，并把模型访问接入 Google Cloud 的身份、网络和治理体系。

来源：https://cloud.google.com/blog/products/ai-machine-learning/claude-at-scale-on-google-cloud-frontier-ai-built-for-enterprise-production

来源：https://6sense.com/newsroom/6sense-launches-mcp-server-bringing-proprietary-gtm-intelligence-into-any-ai-agent/

## 6. QuantumByte：用独立验证 Harness 约束 AI 应用生成

QuantumByte 是一个开源应用构建引擎，从自然语言意图生成可运行项目，并在每次 Agent 执行后依据业务需求进行只读审计。其验证层将成功、失败和无法确定分开记录，附带证据并支持按差异范围重新验证，以减少“界面能跑但关键需求缺失”的假完成。

来源：https://github.com/QuantumByteOSS/quantumbyte

## 7. Rebuttal Skill：为学术审稿回复增加证据与优先级约束

Rebuttal Skill 将论文回复拆成审稿意见分解、严重度判断、P0-P3 实验规划和结果整合两个阶段，并先判断回复还是重新投稿更合适。它要求所有论点由现有证据支撑，禁止虚构数值或把计划实验写成已完成结果，适配 NeurIPS、ICML、ACL 和 CVPR 等场景。

来源：https://github.com/TobiasLee/Rebuttal-Skill

## 8. Toolcraft：面向 AI 编码 Agent 的可视化创作应用脚手架

来源：https://github.com/pixel-point/toolcraft

## 9. Burrow：在浏览器标签页中运行本地 AI 编码环境

Burrow 在浏览器内组合虚拟文件系统、Shell、Git、Bun 转译与服务预览，并通过 Web Workers 和 Service Worker 模拟开发机运行环境。其编码 Agent 使用 WebGPU 上的本地 Qwen3-0.6B 或 Gemma 读取文件、修改代码和执行命令，工作区保存在 IndexedDB 中且无需远端沙箱。

来源：https://github.com/Dhravya/burrow

## 12. BATHOS：以 Rust 引擎固化多角色 Agent 质量门禁
