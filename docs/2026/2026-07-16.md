## 1. Gemini Agent 平台接入 Parallel 实时网页检索

Google Cloud 将 Parallel Web Search 原生集成进 Gemini Enterprise Agent Platform，为 Gemini 提供带精确引用的实时网页 grounding。开发者可从 Gemini API 或 Agent Studio 调用，并能抽取、缓存搜索数据或把结果交给其他模型，零数据保留选项则面向敏感工作负载。

来源：https://developers.googleblog.com/en/expanding-choice-in-gemini-enterprise-agent-platform-introducing-grounding-with-parallel-web-search/

## 2. Google 提出用模块化转译器构建生产级 Agent 提示词

Google 建议把庞大的系统提示词拆成可复用技能文件，再由转译器解析依赖、检查导入和变量并生成确定性构建产物。该方法把提示词纳入版本控制、静态验证和 CI/CD，使 Agent 对自身指令的修改先以代码变更和评测接受人工审查。

来源：https://developers.googleblog.com/building-scalable-ai-agents-with-modular-prompt-transpilation/

## 3. DeepMind 与 Isomorphic Labs 公布 AI 生物韧性方案

双方把生物韧性工作分为预防、检测和响应：对模型执行威胁建模、评测、缓解与监控，同时向可信研究伙伴开放先进系统。方案还探索将 SynthID 用于标记 AI 生成生物序列、用 AlphaEvolve 优化宏基因组分析，并在疫情出现时加速疫苗和药物设计。

来源：https://deepmind.google/blog/our-approach-to-bioresilience/

## 4. Roblox 发布移动端文生游戏工具 Build

Roblox Build 可在主应用中把一句文本提示变成可试玩的基础游戏，并生成玩法、环境、角色、视觉风格与声音。它与 Roblox Studio 共享后端、模型和聊天历史，后续还将加入测试、分析和实验 Agent，帮助创作者委派调试与增长优化任务。

来源：https://ir.roblox.com/news/news-details/2026/Roblox-Introduces-Build-A-New-Way-to-Create-on-the-Platform/default.aspx

## 5. Google Vids 加入个人数字分身与 Gemini Omni 视频编辑

Google Vids 新增由自拍和语音创建的个人数字分身，并用 Gemini Omni 组合文本提示与参考图生成视频。Omni 还支持逐步修改背景、光照和效果，个人分身与 Google 账户绑定并通过 SynthID 加入不可见水印。

来源：https://tech.yahoo.com/ai/gemini/articles/google-vids-now-lets-star-183254400.html

## 6. bbarit-oss 开源单文件 Rust AI 编码 Agent

bbarit-oss 将终端编码 Agent 打包为跨平台 Rust 单一二进制，支持 15 个以上模型提供商和上千个模型。项目加入并行子 Agent 编排、项目 Wiki、跨会话记忆、语义代码搜索和角色系统，并可自动加载 Claude Code 与 Codex 的 MCP Server 和技能。

来源：https://github.com/bbarit/bbarit-agent-oss

## 7. UXON 用 API 与 MCP 将落地页实验交给 Agent

UXON 提供 API 和 MCP 连接器，让开发者或 Agent 创建落地页、导入外部代码、启动跨域 A/B 测试并跟踪转化。平台以第一方归因连接页面、渠道、联系人和收入事件，并允许自动化流程读取转化优化报告或重复执行代理商账户配置。

来源：https://github.com/alexpilotto/uxon-ai

## 8. OpenClaw Desk Pet 用桌宠呈现 Agent 实时状态

OpenClaw Desk Pet 是原生 macOS 桌宠，用不同猫咪动作展示 Agent 的思考、工具调用、完成和休眠状态。用户还可从桌面直接发送快捷任务或自然语言指令，并在统一配置页管理 OpenClaw 路径、Gateway 与按钮。

来源：https://github.com/LeoZhaorx/openclaw-desk-pet

## 9. Codex AutoSkin 从图片自动生成桌面端主题

Codex AutoSkin 通过 Chromium DevTools Protocol 给 Windows Codex 桌面端注入主题，不修改官方文件且可一键还原。它能从任意图片自动取色生成 manifest 主题，也提供面向 Agent 的规范，让 Codex 或 Claude 按裁剪、配色和截图验收流程继续精修。

来源：https://github.com/Finderchangchang/codex-autoskin

## 10. X Post Scheduler 串联摘要、AI 海报和社交发布

X Post Scheduler 把文章抓取、评论式摘要、配图生成与 Buffer 排期组合成 Claude Code Skill，并可调用 gpt-image-2 生成海报底图后用 Pillow 精确叠加中文。长文流程会把代码块和表格转成图片或列表，再通过 Typefully 建草稿，并保留发布前人工确认与敏感内容门禁。

来源：https://github.com/cxjwin/x-post-scheduler

## 11. AgentSmith 提供跨模型的通用 Agent Harness

AgentSmith 将验证、记忆、安全约束和交接规则拆成通用核心与软件、研究、设计、文档等工作类型配置。安装脚本为 Claude、Codex、Gemini 等可读取项目指令的 Agent 组装运行规范，并用真实项目检查、失败测试和审查门禁约束交付。

来源：https://github.com/PromptPartner/agentsmith

## 12. Game Copycat 将截图或一句提示变成原创网页游戏

Game Copycat 是面向 Claude Code、Codex 和其他兼容 Agent 的小游戏制作技能，可从游戏名、链接、截图或一句描述生成原创 Web 游戏。流程强调可证明关卡、浏览器实测、视觉打磨和原创品牌，仓库同时提供由单条提示端到端生成的示例游戏源码。

来源：https://github.com/hellokaton/game-copycat

## 13. LibreCanvas 开源多模态 AI 创作画布

LibreCanvas 是浏览器端 BYOK 无限画布，可在同一项目中执行文本、图像、视频和语音生成，并接入 OpenAI 兼容接口。画布支持多图参考、局部重绘、结果分支与血缘连线，媒体和密钥分别保存在 IndexedDB 与 localStorage，项目本身无需服务器。

来源：https://github.com/Mr-Salticidae/libre-canvas

## 14. LexFlow 面向中文法律场景本地化 AI 工作台

LexFlow 在开源法律平台基础上适配中文律师和法务工作流，提供助理对话、合同与尽调分析、表格化审查、文档版本和项目协作。其数据层可通过 PocketBase 本地部署，默认适配 DeepSeek，并预置北大法宝 MCP 连接器用于法规、案例和引用核验。

来源：https://github.com/jwangkun/LexFlow

## 15. Hypruse 为 Wayland 桌面提供 Agent Computer Use

Hypruse 是面向 Hyprland 的 MCP Server，通过 compositor IPC、截图和原生 Wayland 协议向 Agent 暴露窗口、工作区、鼠标与键盘控制。它优先返回可定位窗口的语义树，在需要视觉判断时再截图，并用事件等待替代多步骤自动化中的固定延时。

来源：https://github.com/IlyasKhallouki/hypruse

## 16. AI Workflows 在 Delphi 中可视化串联生成模型

AI Workflows 为 Delphi FireMonkey 提供 Replicate 模型节点，开发者可在 LiveBindings Designer 中用连线构建图像生成、放大等多阶段流程。组件从模型 JSON Schema 自动生成类型化输入，并在上游结果到达后触发下游节点，同时保留等价的代码式管线接口。

来源：https://github.com/FMXExpress/AI-Workflows

## 17. Axion Designer 用机械验证守住 AI 生成界面风格

Axion Designer 将产品理解、视觉方向探索、真实像素检查和迭代封装成 Agent 设计技能，并用 Taste Lock 保存获批的语义令牌与视觉身份。其验证层检查对比度、无障碍、源码漂移和截图证据，再把结果写入哈希链账本，避免后续 AI 修改悄然偏离设计方向。

来源：https://github.com/fingertipshq/axion-designer

## 18. 16 Eyes 用多重反证降低 AI 安全审计误报

16 Eyes 可对完整仓库或 PR 差异执行 AI 安全审计，先根据技术栈和架构动态设计调查视角，再让独立验证者重新读取代码核查每项发现。高影响问题还会交给多个对抗审查者尝试推翻，只有经受住反证的结果才进入最终报告。

来源：https://github.com/kigiela/16-eyes
