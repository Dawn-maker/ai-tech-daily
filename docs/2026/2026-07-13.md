## 1. SceneSmith：多 Agent 自动生成机器人训练用 3D 场景

MIT CSAIL 与 Toyota Research Institute 发布 SceneSmith，用设计、批评和编排三个视觉语言 Agent 分阶段构建可直接加载到物理模拟器的室内 3D 环境。系统生成了 1300 多个高密度场景，并能为物体补充质量、摩擦和惯性等属性，帮助机器人在真实部署前获得更丰富的视觉与交互训练数据。

来源：https://news.mit.edu/2026/ai-agents-create-virtual-playgrounds-to-help-robots-get-crucial-training-data-0713

## 2. MIT 提出无需生成非法图像的有害模型审计方法

研究团队用 Gaussian probing 分析 LoRA 适配器对模型内部表示的改变，在不运行完整生成流程、不产生违法输出的情况下判断模型是否被专门微调为生成有害图像。实验对已知 CSAM 专用适配器实现了 100% 识别率，为模型托管平台提供了可扩展的上传前安全筛查思路。

来源：https://news.mit.edu/2026/new-method-keeps-kids-safe-from-illegal-ai-generated-content-0713

## 3. CAST AI MCP 服务新增强制认证请求头

CAST AI 文档宣布其远程 MCP 服务自 7 月 13 日起要求所有请求携带 `x-readme-auth: castai-mcp` 请求头，旧配置若不更新会被拒绝。该服务让 Claude Desktop、Cursor 和 Windsurf 等客户端通过 MCP 直接访问 CAST AI API，此变更会影响现有 Agent 集成的连接配置。

来源：https://docs.cast.ai/reference/mcp

## 5. mimic：从真实应用流量自动生成 Python 客户端

mimic 通过 mitmproxy 捕获用户自己的应用请求，提取认证信息与接口结构，再让 Claude 生成可编辑的 Python 客户端。它把移动应用的多步骤 API 调用封装为普通方法，适合为个人自动化或 Agent 快速构建缺少公开 SDK 的工具接口。

来源：https://github.com/littledivy/mimic

## 6. Pixel Art Fixer：修复 AI 图像中的伪像素网格

Retro Diffusion 开源 Pixel Art Fixer，用传统图像处理把生成器、放大器或有损压缩产生的“伪像素画”还原成真正对齐的低分辨率像素网格。项目结合自相关、游程、频谱与重建评分等多种检测器，可处理非整数像素尺寸、网格漂移、抗锯齿和 JPEG 块效应。

来源：https://github.com/Retro-Diffusion/pixel-art-fixer

## 7. TradingView MCP：让编码 Agent 操作本地桌面图表

TradingView MCP 通过 Chrome DevTools Protocol 连接已登录的 TradingView Desktop，为 MCP 客户端提供读取图表、切换代码与周期、获取 OHLCV 数据和截图等 12 个类型化工具。所有处理在本地完成，并附带诊断命令和严格参数校验，降低桌面自动化接口漂移带来的排障成本。

来源：https://github.com/pueschel88/Tradingview-MCP

## 8. RK3506 Voice Robot：在 32 位嵌入式板上运行语音助手前端

该项目用纯 C 在 Rockchip RK3506 开发板上实现按键录音、Wi-Fi 上传、远程 Agent 调用和 LCD 回答显示。运行时采用单线程事件循环与全静态内存分配，面向资源受限设备探索低开销的语音 AI 交互终端。

来源：https://github.com/UIseries/ai-robot

## 9. Swift AI SDK：统一苹果端侧模型与云模型调用

Swift AI SDK 为 iOS 和 macOS 提供供应商无关的 Swift 6 接口，统一 `generateText`、流式输出、结构化对象生成和工具调用循环。开发者可以在 Apple 端侧 Foundation Models、OpenAI 和 Anthropic 后端间切换，并复用 Vercel AI SDK 的 UI 消息流协议。

来源：https://github.com/zaidmukaddam/swift-ai-sdk

## 10. Claude Master：让强模型负责规划与复核、低成本模型执行

Claude Master 是面向 Claude Code 和 OpenCode 的委派插件，由 Claude 先编写实现规格，再把机械性编码工作交给 Codex、OpenCode、Pi 或 Pythinker。执行完成后主模型读取差异和验证输出再决定是否接受，形成“规划—实现—复核”分层工作流。

来源：https://github.com/Pythoughts-labs/claude-master

## 11. RouterClaw：在 32MB 内存的 OpenWrt 路由器上运行自治 Agent

RouterClaw 用 Go 为老旧 OpenWrt 路由器实现常驻 Agent，支持任务拆解、工具链调用、Telegram、Google Workspace、网页访问、Shell 与 Wake-on-LAN。项目针对 MIPS 和极小内存环境编译，展示了把家庭自动化 Agent 下沉到低成本边缘设备的路径。

来源：https://github.com/root643/routerclaw

## 12. Auto Video Gen：把文章或 GitHub 页面自动制作为竖屏短视频

Auto Video Gen 将一个 URL 转换为适配 TikTok、Reels 和 Shorts 的 1080×1920 视频，流水线包含脚本生成、越南语 TTS、字幕、动效模板和最终渲染。项目提供 Claude Code 技能入口，并支持 Vbee、ElevenLabs 与 LucyLab 等语音服务，面向可复用的内容创作工具链。

来源：https://github.com/Cuongyd196/auto-video-gen

## 13. Basically Agent Evals：演示 LLM 评测为何会给出误导性高分

该项目提供一套可运行的 TypeScript 课程，从确定性断言逐步扩展到 LLM-as-judge、RAG 分段评估和诊断控制台。它重点展示位置、长度、迎合和自偏好等裁判偏差，并用人工校准、细粒度 probes 与 CI 回归样例改进评测可信度。

来源：https://github.com/TejasQ/basically-agent-evals

## 14. Launchworthy：为 AI 生成应用提供固定口径的上线审计

Launchworthy 是一个 Claude Code 技能，按前端体验、后端数据、认证安全、基础设施和运维恢复五个域审计由 AI 工具构建的应用。它要求无法验证的检查不得判为通过，并输出带严重级别、证据状态和具体修复路径的可重复评分报告。

来源：https://github.com/Wunderlandmedia/launchworthy

## 15. Grok Privacy Guard：检测并阻止 CLI 静默上传整个代码库

Grok Privacy Guard 通过配置开关、二进制指纹和每次启动自检，阻止 Grok CLI 上传完整代码库、Git 历史与会话 trace。工具还能只读检查历史日志中的上传迹象，并明确提示其保护范围不包括用户主动交给模型读取的文件。

来源：https://github.com/LeifDiao/grok-privacy-guard

## 16. Claude Code Flow Visualizer：把 Agent 配置渲染成交互式关系图

该工具解析 `CLAUDE.md`、子 Agent、技能、命令、钩子、调度器和 MCP 配置，在浏览器中生成完整的委派与依赖图。它还检测缺失引用、循环委派、孤立技能和拥有写权限的审查 Agent 等配置问题，所有解析均可在客户端完成。

来源：https://github.com/oyekamal/claude-code-flow-visualizer

## 17. Case File：带页码、哈希与证据 ID 的本地法律文档 MCP

Case File 将民事案件 PDF 转换为本地 SQLite/FTS5 检索工作区，为每段内容保存稳定 `evidence_id`、页码和哈希，并要求最终陈述通过引用验证。扫描页 OCR 采用显式审批与调用上限，案件目录则限制在授权本地路径中，突出法律 Agent 的来源可追溯性。

来源：https://github.com/carlosvictorodrigues/case-file

## 18. BetterWright：为浏览器 Agent 增加策略、凭据与完成证据

BetterWright 在 Playwright 之上提供跨轮次持久浏览器、逐请求网络策略、加密且按来源隔离的凭据库，以及可供 Agent 引用的证明截图。模型生成的浏览器代码在受限环境中执行，默认阻止云元数据与私有网络访问，降低 DNS 重绑定和秘密泄露风险。

来源：https://github.com/CuriosityOS/betterwright

## 19. Gazebo MCP：让 AI Agent 控制机器人仿真世界

Gazebo MCP 把世界、模型、位姿、暂停和单步执行等 Gazebo/gz-sim 操作暴露为 MCP 工具。它同时提供无需安装 Gazebo 的离线 mock 和可选实时桥接，便于在 CI 中验证机器人 Agent 的工具调用逻辑。

来源：https://github.com/mergeos-bounties/gazebo-mcp

## 20. RViz MCP：让 Agent 配置和检查 ROS 可视化

RViz MCP 为 Agent 提供显示树、固定坐标系、视角、面板、配置读写和截图等 RViz2 操作。项目同样支持离线 mock 与实时桥接，使 ROS 可视化自动化可以在没有完整机器人环境时测试，再切换到本地 RViz 实例。

来源：https://github.com/mergeos-bounties/rviz-mcp
