# AI 技术热点日报｜2026-07-10

## 1. Google Cloud 向所有用户开放 AlphaEvolve

Google Cloud 在 7 月 10 日把 AlphaEvolve 从前沿研究能力推进到面向所有用户可用，核心定位是让 Gemini 系列模型协同搜索、改进和验证复杂算法。它不只是一个“会写代码”的助手，而是把程序生成、候选迭代和结果评估整合成闭环，用于数学、优化和工程问题。对开发者和研究团队来说，这意味着算法发现型 AI 正开始从实验室工具进入云平台产品层。

来源：https://cloud.google.com/blog/products/ai-machine-learning/alphaevolve-now-available-for-everyone

## 2. Google Cloud 发布系统化 agent 评测方法

Google Cloud 同日发布 agent 评测方法论，强调评估不能只看单次任务成功率，而要覆盖任务分解、工具调用、轨迹质量和生产环境稳定性。文章把 agent 评测拆成基准集设计、观测指标、失败归因和线上反馈回路几个层面。对正在把 AI agent 推向生产的团队来说，这套方法比单纯刷 benchmark 更接近真实落地需求。

来源：https://cloud.google.com/blog/products/ai-machine-learning/a-methodical-approach-to-agent-evaluation

## 3. AWS 详解在 SageMaker HyperPod 上做预填充与解码解耦

AWS 7 月 10 日发布了在 SageMaker HyperPod 上实现 LLM inference `disaggregated prefill and decode` 的方案，重点解决长上下文场景下预填充与逐 token 解码资源需求不一致的问题。方案把两段流水拆开调度，以提高 GPU 利用率并降低延迟抖动。对做高吞吐推理服务的团队来说，这是云厂商开始把更细粒度的推理系统优化公开产品化。

来源：https://aws.amazon.com/blogs/machine-learning/disaggregated-prefill-and-decode-for-llm-inference-on-sagemaker-hyperpod/

## 4. AWS 推出 Nemotron 3 的无服务器微调路径

AWS 同日给出基于 SageMaker AI serverless model customization 微调 NVIDIA Nemotron 3 模型的完整路径，目标是把推理型大模型的定制门槛降到更轻的资源形态。重点在于把训练作业、模型托管和部署流程统一到托管服务里，而不是要求团队自己维护复杂训练集群。对企业模型定制场景来说，这会进一步推动“推理模型专项微调”成为标准能力。

来源：https://aws.amazon.com/blogs/machine-learning/fine-tune-nvidia-nemotron-3-models-with-amazon-sagemaker-ai-serverless-model-customization/

## 5. AWS 用 Unsloth 方案推动量化模型部署

AWS 在 7 月 10 日发布了在 SageMaker AI 上部署量化模型并结合 Unsloth 做优化的实践，关注点是用更低显存和更高性价比承载大模型服务。文章说明了量化模型在托管推理中的镜像、打包和加速流程。对中小团队而言，这类方案让“部署可用的大模型”越来越依赖工程优化而不只是堆算力。

来源：https://aws.amazon.com/blogs/machine-learning/deploying-quantized-models-on-amazon-sagemaker-ai-with-unsloth/

## 6. AWS 把语义层引入 AgentCore 工作流

AWS 发布了结合 Stardog 和 Amazon Bedrock AgentCore 构建 agentic AI 语义层的方法，目标是让 agent 在执行任务时具备更稳定的企业知识访问和约束能力。它强调通过知识图谱、实体关系和语义查询来减少 agent 对非结构化检索的依赖。对企业 AI 来说，这反映出 agent 架构正在从“加检索”升级为“加语义控制层”。

来源：https://aws.amazon.com/blogs/machine-learning/build-a-semantic-layer-for-agentic-ai-on-aws-with-stardog-and-amazon-bedrock-agentcore/

## 7. AWS 强调原生 case management 对 agent 工作流的重要性

AWS 7 月 10 日发布 Amazon Quick Automate 的案例管理能力，重点是让多步骤 agent 工作流具备任务状态、人工介入和审计跟踪。它把 agent 从一次性调用扩展到长期流程编排，更接近企业自动化系统的真实要求。对要把 agent 接到客服、审批和运营流程里的团队来说，这类底层能力比单轮问答更关键。

来源：https://aws.amazon.com/blogs/machine-learning/scaling-agentic-workflows-with-native-case-management-in-amazon-quick-automate/

## 8. KTern.AI 披露基于 Bedrock AgentCore 构建 SAP agent 平台的方法

KTern.AI 在 AWS 官方博客上披露了其把 SAP 场景 agent 平台建立在 Amazon Bedrock AgentCore 之上的工程路径，强调复杂企业软件流程需要长期记忆、工作流编排和多 agent 协作。案例展示的不是通用聊天，而是面向 SAP 运维和迁移的业务代理。对企业软件市场来说，这说明 agent 正加速进入 ERP 这类高门槛系统。

来源：https://aws.amazon.com/blogs/machine-learning/how-ktern-ai-built-agentic-ai-for-sap-on-amazon-bedrock-agentcore/

## 9. Hugging Face 发布 PyTorch Profiling 系列第三篇，聚焦 Attention

Hugging Face 在 7 月 10 日发布 `Profiling in PyTorch (Part 3)`，把分析重点放到 attention 路径的性能瓶颈上。文章从 kernel 级视角解释不同 attention 实现的耗时来源，以及如何借助 profiling 结果指导算子和内存优化。对训练和推理工程师来说，这类内容的价值在于把模型性能调优从经验活变成可观测、可复现的工程流程。

来源：https://huggingface.co/blog/torch-attention-profile
