## 5. Blinkface 用手势框选触发 FLUX.2 实时人像重绘

Blinkface 把 MediaPipe 手部关键点与 FLUX.2 Klein 4B 图像模型结合，用户用双手在摄像头前构成取景框后，即可把框内人像实时转换为动画、赛博朋克等风格，闭合手势还能切换下一种样式。系统把 CUDA GPU 上的 FastAPI 与 Diffusers 推理服务和 Mac 或浏览器遥控端分离，支持文生图、图生图以及经局域网、Tailscale 或 SSH 隧道访问。

来源：https://github.com/xcc3641/blinkface

## 6. Frame Studio 从提示词生成可编译的 Remotion 视频

Frame Studio 先让 Gemini 规划视频结构并生成 React/Remotion 代码，再在沙箱中做 TypeScript 编译，失败时最多自动修复三轮，最后渲染并向浏览器流式返回 MP4。项目把轻量 Next.js 前端与 CPU、内存密集的渲染 Worker 分开部署，并通过队列限制并发、向界面回传排队位置，形成从文本到 1080p 动效视频的完整流水线。

来源：https://github.com/programmersd21/frame_studio

## 7. PVDepth 开源全景视频深度估计模型与数据集

PVDepth 针对等距柱状投影的空间畸变和全景视频中的非线性时间变化，采用几何感知的时空适配生成连续、稳定的深度序列，并同步开放 PanoCARLA 合成 RGB-D 全景视频数据集、训练代码、评测脚本与权重。项目支持单图、图片目录或视频输入，110 帧 1024×512 推理约需 31GB 显存，也可通过顺序 CPU 卸载把需求降到约 10GB，但会显著牺牲速度。

来源：https://github.com/ChuanxinSong/PVDepth

## 8. Accio 将 macOS GUI 操作嵌入普通 Python Agent 流程

Accio Computer Use 允许 Agent 在一个 Python 代码块中混合桌面点击、Shell、应用 CLI、API、文件操作和条件循环，只在确有必要时进入 GUI，从而减少模型逐步规划每次点击的往返。每项桌面操作都会返回刷新后的无障碍树与截图证据，类型签名会在执行前拦截无效调用，成功代码还能封装为可复用函数或工作流；当前为 macOS 14 以上的源码版本。

来源：https://github.com/Accio-Work-Lab/accio-cu

## 9. Capybara 在终端定位 AI Agent 的错误决策链

Capybara 是单文件 Go TUI，可接收 OTLP 轨迹或读取 Claude 会话文件，把提示、工具调用、Token、成本、异常和最终回答保存到本地 SQLite，再标出 Agent 在哪一步偏离。它兼容 OpenTelemetry `gen_ai.*`、OpenInference、OpenLLMetry 和 Vercel AI SDK 等属性约定，并能识别工具失败后仍继续回答、无效检索和未使用结果等常见问题。

来源：https://github.com/tonquoc0407/capybara

## 10. Pinvou Agent 统一本地模型、知识、工具与交付物

Pinvou Agent 是基于 Tauri 的开源桌面工作区，把 OpenAI 兼容或本地 vLLM 模型、MCP、CLI、Skill、长期记忆、向量检索和多步工作流放入同一 Agent 运行环境。系统支持导入 PDF、Office、图片和文本，直接创建或编辑文档、表格与代码并集中预览交付物，同时提供需审查的 Plan 模式和直接执行模式；数据是否离开本机取决于用户启用的模型与连接器。

来源：https://github.com/Pinvou/pinvou-agent

## 11. Triton OpenAI Gateway 补齐多模态生产推理协议

Triton OpenAI Gateway 在 NVIDIA Triton 与 vLLM 前提供 OpenAI 兼容接口，统一处理模型聊天模板、工具调用、文本与视觉对话、Embedding 和重排序，并为图片、视频、音频与 PDF 设置有界预处理路径。网关还加入准入队列、取消、超时、429 背压、Prometheus 指标、OTLP 轨迹和 Kubernetes Helm 部署，但不负责用户认证、外部工具执行或模型权重分发。

来源：https://github.com/Raytorin/triton-openai-gateway

## 12. Switchboard 用本地可解释路由连接 16 家模型供应商

Switchboard 在本机暴露统一的 OpenAI 兼容端点，根据实时健康、延迟、价格和免费额度为每个请求选择模型供应商，遇到限流或故障时自动转向下一候选。每次路由都会把候选得分、排除原因、响应头和成本写入本地数据库，密钥也保存在操作系统应用数据目录，除实际上游调用外不依赖云端控制面。

来源：https://github.com/abeermeer/switchboard

## 13. ChunkLab 用自有问题集比较 RAG 分块策略

ChunkLab 固定 Embedding 模型和检索器，只改变固定长度、递归、语义与结构化分块，使用用户标注的答案片段计算 Recall@5、Hit@5 和 MRR，并输出边界健康、碎片率、表格完整性与 Token 分布诊断。工具默认用本地 BGE Small 模型运行，无需 API 密钥，结果同时生成终端表格、可逐题查看边界的 HTML 报告和供 CI 消费的 JSON。

来源：https://github.com/Ludovico-Chieffallo/chunklab

## 14. paper2sim 让 Agent 生成并运行论文验证实验

paper2sim 接收 arXiv 链接、PDF 或文本，先提取论文最关键的可测试主张，再生成自包含 Python 模拟，在资源受限沙箱内执行、失败修复，并以图表、动画和自然语言结论说明实验是否支持主张。系统按数学、计算机科学和物理选择蒙特卡洛、复杂度测量、符号计算或数值积分等路径，后端使用 FastAPI、Celery 与 Redis，也提供不需密钥的确定性模拟流程用于本地验证和 CI。

来源：https://github.com/ashw1nkumars/paper2sim

## 15. F3-OpenScience 为科研 Agent 加入独立确定性审计

F3-OpenScience 在文献检索、分析与写作流水线的输出前加入独立验证层，以确定性规则核验引用、数字和可追溯证据，并把被拦截的错误沉淀为可复用验证经验，而不是依赖生成模型自我约束。项目明确标为研究原型：验证组件和可达性机制虽有测试，但尚未用真实 LLM 端到端产出论文，桌面应用、Docker 编排、安全性与规模性能也未完成实测。

来源：https://github.com/MedocMay/F3-OpenScience

## 16. Agent Report Card 评估工具调用是否真正贡献答案

Agent Report Card 在传统时延轨迹之外，为每个 Agent 决策计算工具结果利用率、相邻推理连贯度、有效与浪费步骤、文字置信信号和整体策略类型，并把报告保存到 SQLite。当前实现用二元词组重叠和正则启发式判断结果是否进入答案，代码清晰且可嵌入现有循环，但作者也明确指出这些指标不是因果真值，生产使用应以 Embedding、LLM 裁判或人工标注校准。

来源：https://github.com/gmaheshraju/agent-report-card

## 17. Xcode 清理 Skill 用双重审批保护不可逆数据

Xcode Disk Cleanup Skill 先只读扫描 DerivedData、模拟器与运行时、预览设备、归档、dSYM、DeviceSupport、缓存、诊断日志和旧安装包，为每项候选报告稳定 ID、精确大小、风险、再生成成本与证据。只有用户逐项批准后脚本才会执行，并在删除前复核路径身份和运行中进程；普通文件默认移入废纸篓，模拟器等不可逆操作与未确认备份的发布归档需要额外确认。

来源：https://github.com/AvdLee/Xcode-Disk-Cleanup-Agent-Skill

## 18. PradaSlides 按运行能力路由演示文稿制作流程

PradaSlides 是面向 Agent 的演示文稿 Skill，会把简短需求或资料整理成沟通简报、证据结构、叙事顺序、视觉方向和可编辑页面计划，再按运行环境能力输出 PPTX、HTML、PDF 或纯规划稿。工作流显式区分文本、视觉理解、图像生成、视频生成与委派能力，并把字体、对比度、裁切、媒体角色、舞台可读性和最终渲染检查纳入交付验证。

来源：https://github.com/prodigeproject/pradaslides
