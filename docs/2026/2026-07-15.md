## 1. GPTBots 为企业 Agent 接入 Modellix 图像与视频生成工具

GPTBots.ai 在工具生态中加入 Modellix 驱动的图像和视频生成能力，让 Agent 可把文本、参考图和既有素材转成营销视觉、产品讲解与培训内容。新工具能与 FlowAgent、工作流、知识库和 API 串联，使多模态资产生成进入可重复的企业流程，而不只是独立的创作页面。

## 2. GoDaddy 推出面向 AI Agent 的域名开发平台

GoDaddy Developer Platform 把域名搜索、购买、DNS 配置和管理整合进新 API 与命令行工具，目标是让编码助手和自动化 Agent 不离开开发环境即可完成域名生命周期操作。平台用 OAuth 细分授权范围，并在购买和不可逆操作前要求明确确认，以限制 Agent 的执行权限和支出风险。

来源：https://www.prnewswire.com/news-releases/godaddy-reimagines-the-domain-experience-for-developers-302825597.html

## 3. gbro-collage-broll：用三阶段审批生成纸拼贴 B-roll

gbro-collage-broll 把约五秒口播压缩成视觉隐喻，再生成半调纸拼贴风格的竖屏动画。技能将隐喻确认、静帧确认和视频生成拆成三道门禁，最后调用 Gemini Omni Flash 完成首尾帧动画，并以逐秒抽帧和首尾帧对照执行质量检查。

来源：https://github.com/pyang5166/gbro-collage-broll

## 4. SuperMap 构建具身 AI 的时空语义记忆

SuperMap 是面向视觉语言导航的训练无关时空 SLAM 系统，将高频几何定位与异步开放词汇感知融合成可查询的 4D 场景图。系统持续记录物体身份、位置和时间变化，可在遮挡和环境变化后保持长期记忆，并为机器人空间推理与行动提供语义地图。

来源：https://github.com/superxslam/SuperMap

## 5. Code Humanizer 检测并清理 AI 生成代码的结构性冗余

Code Humanizer 将 AI 编码常见问题整理为 16 类模式，包括重复帮助函数、无意义抽象、宽泛异常捕获、属性探测链和低价值测试。该 Agent Skill 先建立测试基线，再以行为保持为约束逐项清理，并要求每批修改后运行相关测试以降低重构风险。

来源：https://github.com/LeonardNJU/code-humanizer

## 6. OTel Research 比较三种 AI 编码 Agent 的本地遥测

该研究用本地 LGTM 栈收集 Claude Code、Codex 和 GitHub Copilot CLI 的 OpenTelemetry 数据，比较请求、Token、缓存、工具活动与成本可见性。项目还用规范化侧车把不同 logs、metrics 和 traces 映射到统一字段，从而在同一 Grafana 仪表板中跨工具分析使用情况。

来源：https://github.com/doggy8088/otel-research

## 7. Scrapewright 将自然语言抓取需求发布为本地 API

Scrapewright 通过 Chrome 扩展和 Node.js Native Messaging Host，让大模型分析真实浏览器页面、生成抓取步骤并返回结构化 JSON。生成的步骤图可保存为带 JSON Schema 的 HTTP 服务，支持登录态页面、动态 SPA、iframe 和多步交互，同时让数据与浏览器凭据留在本机。

来源：https://github.com/singhand-labs/scrapewright

## 8. AI Coding Usage Card 汇总多种编码助手的本地消耗

AI Coding Usage Card 从 ccusage 可识别的本地日志汇总 Claude Code、Codex、Gemini CLI、Copilot CLI 与 OpenCode 的 Token 和 API 等价成本。工具在本机生成多种尺寸的 SVG 卡片及 26 周热力图，可自动提交到 GitHub 个人资料仓库，不需要外部账户或 API 密钥。

来源：https://github.com/Baek-Seunghyun/ai-coding-usage-card

## 9. InkPaw 为文档 Agent 提供确定性 DOCX 工具链

InkPaw 用结构化文档模型把内容生成与 Word 排版细节分离：模型负责文本和结构，确定性代码负责编号、样式、交叉引用与 OOXML 正确性。项目提供 MCP Server、本地 Agent Playground、DOCX/PDF/逐页 PNG 渲染和三层验证，支持在多轮编辑中稳定定位并修改节点。

来源：https://github.com/syouro/InkPaw

## 10. Media Transcribe 统一多平台音视频转录流程

Media Transcribe 将抖音、B 站、TikTok、微博、YouTube、播客、RSS 和本地音频纳入同一个 Agent Skill，并输出结构统一的 Markdown。它按来源选择 SenseVoice-Small 或 faster-whisper，支持批量队列、断点续跑和英文内容中文翻译，同时明确只处理公开且获授权的媒体。

来源：https://github.com/loqz99156/media-transcribe

## 11. Pixel Bench 建立像素画重建的开放评测基准

Pixel Bench 对原生 1x 像素画施加非整数缩放、模糊、JPEG、噪声、网格漂移和 AI 放大器伪影，再评估各类重建方法。基准从原生尺寸、颜色、像素位置与网格对齐等维度打分，并用版本化失真套件和统一结果格式支持跨工具复现比较。

来源：https://github.com/Retro-Diffusion/pixel-bench

## 12. Continual Learning Terminal Bench 评测 Agent 优化器能否持续积累

该项目发布 Terminal-Bench 2.0 上的持续学习实验产物，对比 GEPA、Meta Harness 和 RELAI-VCL 在两阶段新增任务后的优化表现。仓库保留每种方法的基线、第一阶段和第二阶段 Agent 快照，覆盖只改提示词、改 Harness 代码以及同时搜索工具、记忆、技能和工作流等不同优化空间。

来源：https://github.com/relai-ai/Continual-Learning-Terminal-Bench

## 14. React AI ChatKit 开源可定制对话界面组件

## 15. ai-cli 用单文件 C 程序连接本地模型与 Shell

ai-cli 将自然语言请求发送到 OpenAI 兼容的本地模型服务，并把返回的 Shell 操作放入交互式编辑界面。用户可在执行前检查、修改或拒绝命令，程序本身保持单个 C 文件和零依赖设计，并支持用本地 Markdown 文件保存可选会话记忆。

来源：https://github.com/vkataev/ai-cli
