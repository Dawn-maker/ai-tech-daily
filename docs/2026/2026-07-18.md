# AI 技术热点日报｜2026-07-18

筛选口径：只收录北京时间 2026-07-18 00:00-23:59 发布、创建或实质更新的 AI 技术内容。技术新闻优先，关注模型、API、开源项目、论文进展、AI agent、图像/视频/多模态生成、算力基础设施、开发者工具和安全评估。每条 `来源` 都指向具体页面。

## 1. Handdraw Story Video 自动制作线稿渐变上色视频

Handdraw Story Video 把 7 至 9 幅手绘故事母图编排成 35 至 45 秒竖屏短片，先逐步显现黑白线稿，再沿同一方向填入低饱和色彩。工具用本地处理保证线稿与彩色图严格对齐，并通过 HyperFrames、GSAP 和 FFmpeg 完成渲染，还提供故事配置、素材与时长校验及 Codex Skill。

来源：https://github.com/xiejunjie524/handdraw-story-video

## 2. Voice Clone Lab 开源 Qwen3-TTS 本地声音克隆管线

Voice Clone Lab 用 5 至 15 分钟授权语音完成提取、清洗、切分、转写、人工校对和 Qwen3-TTS 微调，也支持单片段零样本试音。项目提供 CLI 与 Web UI，训练数据和模型均留在本机，并明确要求仅克隆本人声音或获得书面许可的声音。

来源：https://github.com/tetsuo-ai/voice_clone_lab

## 3. Reeled In 用神经模型预评估短视频版本

Reeled In 接收多个 Reel 或 TikTok 剪辑版本，用 Meta TRIBE v2 预测视觉、听觉、语言、运动和默认模式五类脑反应，再比较峰值、持续参与度与结尾留存。其语音 A/B 流程还可用 ElevenLabs 为同一脚本生成不同旁白，项目同时说明脑激活只是方向性参与度代理，不能等同于销量或传播效果。

来源：https://github.com/sxnnywu/reeled-in

## 4. HY-Motion 扩展将文生动作重定向到 Mixamo 角色

该 ComfyUI 扩展接收 HY-Motion 输出的人体动作，通过后台运行 Blender 把动作重定向到 Mixamo T 姿势角色。它会根据体型修正根节点位移、接地和手臂间隙，并分别导出带位移的交付 FBX 与便于 Preview3D 查看原地动作的 GLB。

来源：https://github.com/zasuko/HY-Motion_Extensions_Text-to-Motion

## 5. WhimprFlow 用本地 Whisper 与 Qwen 清理语音输入

WhimprFlow 是 Tauri 构建的跨平台按键说话工具，先在设备上用 Whisper 转写，再用本地 Qwen3-4B 清除语气词、处理口头改写并整理标点和列表。项目还提供个人词典、原文回退、可选云端清理与系统密钥库，当前明确定位为需要进一步测试和加固的概念验证。

来源：https://github.com/Blueturboguy07/WhimprFlow

## 6. Embinder 让 Agent 能力随应用当前界面动态装载

Embinder 通过 React Hook 把组件显示的状态和可调用处理函数注册给应用内 Agent，组件卸载后相应上下文与能力也自动移除。其目标是让模型只看到当前页面可用的少量操作，并以服务端批准门禁和审计层保护高风险调用，而无需额外连接器或独立工具服务器。

来源：https://github.com/celesnity/Embinder

## 7. gMAS 用动态图实现去中心化多 Agent 编排

gMAS 基于 rustworkx 把 Agent、通信边和图算法作为一等对象，允许运行时修改拓扑，并让每个节点保有独立状态。框架支持强连通分量与拓扑调度、同步和异步流式执行、请求预算、回调，以及把图转换为 PyTorch Geometric 结构进行路由研究。

来源：https://github.com/frontier-ai-next/gMAS

## 8. Umbrella 用分阶段控制面治理深度 Agent

Umbrella 把研究、规划、执行、审查和验证拆成带工具权限、技能和退出标准的阶段清单，并为每个阶段启动工作 Agent 与只读监督者。控制面还可让多个 Agent 头在同一阶段竞争，再依据证据选出结果，同时通过权限包、持久记忆和失败反思约束长期运行。

来源：https://github.com/frontier-ai-next/Umbrella

## 9. Lethyx 提供框架无关的 Agent 长期记忆层

Lethyx 将记忆写入、检索、遗忘、证据晋升、上下文构建与图导出封装为一组稳定 API，不接管宿主 Agent 的路由和决策。它可选择 MemPalace、Hindsight 或 Mnemosyne 作为持久后端，并提供 LangGraph 适配器、MCP Server 和记忆场景评测工具。

来源：https://github.com/frontier-ai-next/Lethyx

## 10. m365-native 将授权 Copilot 会话转换为兼容 API

m365-native 为用户有权访问的 Microsoft 365 Copilot 会话提供 OpenAI Chat Completions、Responses 与 Anthropic Messages 兼容接口。网关支持流式响应、多模态输入、工具调用转换、会话连续性和上游图片事件解析，并通过 PKCE 授权、密钥哈希与本地默认监听降低凭据暴露风险。

来源：https://github.com/uefi2333/m365-native

## 11. Cache Hunter 诊断 LLM 前缀缓存失效

Cache Hunter 在 Agent Harness 与 OpenAI 兼容端点之间部署透明代理，把请求、响应、耗时和 Token 用量记录到 SQLite。分析工具利用前缀哈希、对话链与延迟趋势定位缓存失效，还支持检查上下文连贯性并导出 CSV 进一步可视化。

来源：https://github.com/co-l/cache-hunter

## 12. CC Cache Playbook 总结长对话上下文增量策略

CC Cache Playbook 针对自建网关连接 Claude Code 订阅的长会话，提出用用户消息内容哈希判断连续性，并只重发发生变化的动态上下文分段。文档还给出墓碑删除、定期重锚、按 Token 加权命中率、摘要罐头检测和失败上游切换等工程防护，目标是保持稳定前缀和高缓存命中率。

来源：https://github.com/51enuxu/cc-cache-playbook

## 13. Design Judge Skills 用证据约束设计奖申报 Agent

Design Judge Skills 把获奖案例检索、设计评价、奖项匹配、申报文字准备和提交终检拆成五个独立技能。工作流要求奖项规则回到当届官方页面核验，将用户事实、模型推断与待确认项分开标记，并用透明量表解释匹配结果而不预测获奖概率。

来源：https://github.com/SeanJ1ang/design-judge-skills

## 14. ICM Architect 用目录结构承载 Agent 流程状态

ICM Architect 将业务过程或现有资料库重构为可解释的文件夹工作区，用编号目录表达顺序、层级表达上下文范围、Markdown 文件保存状态。它提供管线、组合管线、记录库、知识包与上下文图五种结构，并通过无记忆 Agent 的“步行测试”验证能否仅靠文件完成定位、行动和汇报。

来源：https://github.com/RinDig/icm-architect

## 15. Reset! 在 macOS 统一监控多个编码 Agent 额度

Reset! 是 SwiftUI 菜单栏工具，可读取本机登录态并显示 Codex、Claude Code、Antigravity 和 Cursor 的五小时及每周额度。它还能用 iCloud 协调多台 Mac 的通知职责，并通过自建 Telegram Bot 推送额度变化与重置提醒。

来源：https://github.com/EEliberto/Reset-macOS
