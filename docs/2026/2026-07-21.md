## 2. Grabette 开放低成本机器人操作数据采集系统

Pollen Robotics 发布 Grabette，用手持夹爪、双摄像头、IMU 和编码器同步记录人类操作，再通过浏览器中的 SLAM 流程转换为标准 LeRobot 数据集。项目开放硬件、采集服务和处理管线，约 490 欧元的采集端可在没有机器人和遥操作台的情况下收集 6-DoF 演示数据，并提供用 200 条示范训练扩散策略的完整样例。

来源：https://huggingface.co/blog/grabette

来源：https://press.spglobal.com/2026-07-21-S-P-Global-Launches-Adaptive-Retrieval%2C-Giving-Customers-a-New-Way-to-Access-Data-Across-AI-and-Agentic-Workflows

## 6. Agent Notch 在 MacBook 刘海旁显示编码 Agent 状态

Agent Notch 通过进程、打开文件和本地会话转录识别 Claude Code 与 Codex 的运行、空闲和完成状态，无需 Hook、API 或账号。它把主会话与子 Agent 按提示词分组展示，并用各自的动态角色和完成提示占用 MacBook 刘海旁的小块区域，点击后可查看模型、会话和输出片段。

来源：https://github.com/realfishsam/agent-notch

## 9. BossConsole 开放跨平台 Agent 操作台

## 10. Story to Handdrawn Video 自动生成手绘日记漫画动画

Story to Handdrawn Video 将中文故事或有序图片拆成竖屏分镜，生成手写字幕以及“文字到黑白画稿再到彩色插画”的揭示动画，并用 Remotion 输出 H.264 视频。项目同时提供渲染器和可供 Codex 等 Agent 安装的 Skill，支持快速预览、正式分辨率和可选卷页转场，便于后续配音与剪辑。

来源：https://github.com/gnipbao/story-to-handdrawn-video

## 11. Style Pack Skill 从参考图提取可复用视觉语法

Style Pack Skill 从电影剧照、视频截图或参考图文件夹提取色彩、光影、构图、镜头、材质和节奏，而不复制原作具体场景与元素。它将新剧本拆成图像与视频提示词，并为每组强制生成标注版、纯色版两套 1920×1080 色卡及结构化色值 JSON，再用验收器检查完整性。

来源：https://github.com/irenerachel/style-pack-skill

## 12. pgContext 把混合 AI 检索直接放进 PostgreSQL

pgContext 是 PostgreSQL 17 扩展，在数据库内部提供稠密向量、全文混合检索、元数据过滤 HNSW、分组和分面等能力。它让原表继续承担 MVCC、ACL、行级安全、备份和复制，并对近似召回结果按当前可见行精确重排，避免维护会漂移的第二份检索数据。

来源：https://github.com/Evokoa/pgContext

## 13. PlotCraft 将自然语言转换成可编辑科研矢量图

PlotCraft 调用兼容 OpenAI 接口或 Gemini 的模型生成 Matplotlib 代码，再在禁网、限制导入和 60 秒超时的子进程沙箱中渲染 SVG。它内置八类科研图模板、本地加密 API 密钥、代码编辑与无额外模型调用的重新渲染，并用 SQLite 保存提示词、代码、结果和错误历史。

来源：https://github.com/jaychouchannel/PlotCraft

## 14. Search Eval 建立可复跑的网页搜索评测闭环

Search Eval 把标注查询、多后端抓取、盲化位置互换评审、目标 URL 命中和端到端 Agent 正确率串成统一框架。数据与运行结果均用 JSONL 和 Git 版本化，既可构建内部基准，也能用官方评分器运行 SimpleQA 与 FreshQA，并生成切片报告和回归差异。

来源：https://github.com/Octen-Team/search-eval

## 15. OptiMCP 为 Agent 提供可验证的约束决策

OptiMCP 接收结构化预算、排班或资源分配条件，用确定性优化器求解，并由独立验证层逐项检查约束，而不是让大模型自行猜测可行方案。它不内置 LLM，可通过 MCP、OpenAI/Anthropic 函数模式或 LangChain 接入，同时提供精确、启发式回退与验证证书。

来源：https://github.com/ProfessionalQwerty/OptiMCP

## 16. FindandSeek 为本地文件建立 Agent 记忆层

FindandSeek 在 Apple Silicon 上用 MLX、在 Linux 上用 Ollama 本地理解文档、代码、邮件和带 OCR 的图片，并把索引保存到 SQLite 后通过 MCP 提供给助手。检索先返回紧凑事实卡片，必要时再读取相关段落，以减少每次查询重复扫描整库带来的上下文和成本，同时由后台 Worker 与文件监视器持续更新索引。

来源：https://github.com/MunasheChitima/findandseek-engine

## 17. CLEAR-LeWM 提供潜在世界模型的可审计评测层

来源：https://github.com/DavidSunok/CLEAR-LeWM
