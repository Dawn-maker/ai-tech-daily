## 1. OpenAI 提出以“每美元有用智能”衡量 AI 价值

OpenAI 建议把 AI 成效从席位数、活跃用户或每 Token 成本，转向真正完成且达到质量门槛的工作。其计分框架同时追踪有用工作量、成功任务总成本、结果可靠性和规模扩大后的边际价值，并把人工复核、重试与返工纳入成本。

来源：https://openai.com/index/a-scorecard-for-the-ai-age/

## 2. Moonshot 发布 2.8 万亿参数开放模型 Kimi K3

Moonshot 宣布 Kimi K3，总参数量 2.8 万亿，并采用混合专家架构降低单次推理实际激活的参数规模。该模型重点面向编码与 Agent 任务，已提供在线服务，完整权重计划于 7 月 27 日开放。

来源：https://apnews.com/article/0d8a5e268deb11a673f4d444fc597cc5

## 3. Tensor Labs 演示用 Gemini Omni Flash 构建对话式视频服务

教程以 FastAPI 封装视频生成、自然语言迭代编辑和成本台账，输入可组合文本、图片与已有视频。服务保存每次生成版本和用量，让创作者能在同一会话中连续修改镜头，同时把多模态调用成本显式化。

来源：https://tensorlabs.io/articles/build-a-conversational-video-clip-service

## 4. OpenAI Academy 发布 AI 工作流证据教练

Workflow Evidence Coach 把需求文档、提示词、测试、截图、指标和反馈整理成可核验的工作流案例。它要求将结论标记为测量、观察、报告、估算或未知，区分相关性与因果性，并只建议证据足以支持的试点或扩展范围。

来源：https://academy.openai.com/public/clubs/champions-ecqup/resources/workflow-evidence-coach-2026-07-17

## 5. Conversation Steganography 用本地模型生成隐写聊天文本

Conversation Steganography 先加密秘密消息，再由本地语言模型把密文编码成看似普通的对话文本，接收方用共享密钥恢复原文。项目提供双人模拟模式验证完整编码链路，但明确标注为仍有局限的概念验证。

来源：https://github.com/nethical6/conversation-steganography

## 6. Klaat Code 用逐请求模型路由降低编码 Agent 成本

Klaat Code 是终端原生编码 Agent，由托管路由模型按任务难度在五档模型层级间选择并可动态升级。客户端还用代码知识图谱查询符号、调用方和影响范围，仓库附带可复现基准以比较准确率、成本与 Token 用量。

来源：https://github.com/KlaatAI/klaatcode

## 7. Pinscope 让 AI 对照数据手册审查电子原理图

Pinscope 将网表与 BOM 建成器件—网络图，并从数据手册中提取引脚、绝对额定值和推荐工作条件。模型按器件检查相邻电路并为发现附上具体手册页码，而 BOM 汇总和电容降额等可确定计算仍由程序完成。

来源：https://github.com/Faradworks/Pinscope

## 8. Datamatic 用 Schema 引导多步骤 AI 数据工作流

Datamatic 通过 YAML 编排结构化生成、迭代、数据变换和命令行步骤，并用 JSON Schema 校验模型输出。它同时支持 Ollama、LM Studio、OpenAI、OpenRouter 和 Gemini，可把视觉分析、合成数据生成与 Hugging Face 数据集接入同一管线。

来源：https://github.com/craftingcodegig/datamatic

## 9. Enterprise-Bench 开放企业 Agent 的生产规模评测

Enterprise-Bench 首版覆盖工程、销售和支持任务，用模拟 Salesforce、Jira 与 Google Drive 接口测试 Agent 跨系统检索和分析能力。评测采用答案保持的数据扩容设计与语义评分，区分确定性工具操作、分析任务和更高自主级别。

来源：https://github.com/devrev/enterprise-bench

## 10. EvalCore 把 AI 行为快照测试接入 CI

EvalCore 是配置优先的单二进制评测运行器，可测试 LLM 应用输出与 Agent 的 OpenTelemetry 行动轨迹。其 SQLite 录制回放缓存让 CI 离线复现模型及裁判结果，并可按基线差异、最低得分、成本和预算设置发布门禁。

来源：https://github.com/Eval-core/evalcore

## 11. CredsHound 扫描本机 AI 工具遗留凭据

CredsHound 用产品模板和通用规则查找环境变量、配置文件、Shell 历史及其他凭据载体。除传统云与 DevOps 密钥外，它覆盖 OpenCode、GitHub Copilot CLI、Hugging Face 和 OpenAI 等 AI 开发环境，并能导出 BloodHound 图展示暴露路径。

来源：https://github.com/haxxm0nkey/credshound

## 12. CitizenAgent 集成视觉理解、生图与并行子 Agent

CitizenAgent 是可切换模型的终端助手，分别接入 DeepSeek 对话、Qwen-VL 图像理解和通义万相图像生成。它还能执行本地命令、跨工作目录保留会话，并用最多五路子 Agent 并行处理独立任务。

来源：https://github.com/ElarineNycanis/Citizen-Agent

## 13. Coco Voice 开源完全离线的跨平台语音输入

Coco Voice 用 Tauri 构建桌面语音转文字工具，在本机运行 Whisper 系列 GGML/GGUF 模型或 Parakeet ONNX 模型。它支持全局快捷键、按住说话、无头文件转写，并可选把结果交给兼容 API 做语法修正或行动项提取。

来源：https://github.com/coco-research/Coco-Voice

## 14. TurnCoder 让用户直接编辑 Agent 的完整上下文

TurnCoder 把模型回复、工具参数与执行结果都显示为可编辑的回合气泡，用户可在真正执行工具前修改或跳过操作。工作台支持二十多个模型提供商、同一会话切换模型、多会话并行和移动端远程访问，文件与命令均在本机处理。

来源：https://github.com/nightwindnohen/TurnCoder

## 15. Sidestack 开源浏览器侧栏 Agent 技术栈

Sidestack 组合 Chrome Manifest V3 侧栏扩展、LangGraph 服务和 WebSocket 桥接层，让 Agent 在浏览器界面中对话和调用工具。它可接 Anthropic、Vercel AI Gateway、OpenRouter，也能通过 Ollama 或 llama.cpp 完全本地运行。

来源：https://github.com/lucianfialho/sidestack

## 16. Gemma Coder 将本地模型变成云端 Agent 的代码执行层

Gemma Coder 把规划和审查留给主 Agent，再把具体代码生成交给本地 Gemma 模型，以减少云端生成调用。该技能可用于 Claude Code、Antigravity 和 Codex CLI，并兼容 Ollama、LM Studio、llama.cpp 与 MLX 等本地运行时。

来源：https://github.com/harshdattani23/gemma-vibecoding-skills

## 17. Local AV to Text 在本机完成影音转写与摘要

Local AV to Text 以 FFmpeg 和 Whisper 抽取并转写视频或音频，再可调用本地 Ollama 生成摘要。工具针对繁体中文使用场景优化，强调素材不上传云端，并提供 macOS 与 Windows 的启动方式。

来源：https://github.com/av8d-levelup/local-av-to-text

## 18. Remote Desktop Commander 推出远程 MCP 连接器

Remote Desktop Commander 通过托管 MCP Server 和本机设备代理，让兼容 AI 客户端从远端访问文件系统与终端。连接使用 OAuth 2.0 设备授权，并可在一个账户下配对多台机器；仓库提供公开清单与文档，服务实现本身不开放源码。

来源：https://github.com/desktop-commander/remote-desktop-commander

## 19. Claude Skin 支持用 AI 生图生成跨端主题

Claude Skin 为 claude.ai、Claude Code 终端仪表盘和实验性桌面副本提供统一换肤系统。网页扩展可通过 OpenAI 兼容接口从一句描述生成壁纸并自动取色，再用 `.claudetheme` 文件把主题同步到网页与终端。

来源：https://github.com/lumos-tiamo/claude-skin

## 20. ModelDock 统一发现、缓存和加载本地模型

ModelDock 提供 Python API 与命令行，用可搜索目录发现模型并自动下载、校验和缓存。它不直接执行推理，而是通过可插拔适配器管理 Ollama，后续规划接入 LM Studio、llama.cpp、Jan、GPT4All 和 vLLM。

来源：https://github.com/OpenAgentHQ/modeldock
